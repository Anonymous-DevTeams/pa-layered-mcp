    Workspace Setup#

    Directory structure for efficient work with PA-Layer system.

    Recommended Structure#

    ~/playground/                    # Workspace root
├── mcp-docs/                    # ObsidianVault (documents)
│   ├── .obsidian/               # Obsidian settings
│   ├── ports/                   # Port documents
│   │   ├── auth/
│   │   │   ├── L1-AuthPort.md
│   │   │   ├── L2-LoginPort.md
│   │   │   └── L2-SignupPort.md
│   │   └── users/
│   │       └── L2-ProfilePort.md
│   ├── templates/               # Templates
│   ├── guides/                  # Guides
│   └── .pa-index.yaml           # Index settings
│
├── CodeSpace/                   # Code projects
│   ├── my-backend/              # Kotlin/Spring project
│   ├── my-frontend/             # React project
│   └── my-api/                  # Go API project
│
└── bundles/                     # Shared Bundle storage

~/.pa/                           # PA-Retriever settings
├── config.yaml                  # Main config
├── default.db                   # Index database
├── keys/                        # Signing keys
└── peers.json                   # LAN peer cache

    Benefits#

    1. AI Agent Collaboration#

    • Clear context: AI knows which documents/code to reference
    • Token efficiency: Load only needed documents
    • Consistent structure: AI can navigate predictable locations

    Setup Checklist#

    • Create ~/playground directory
    • Create ~/playground/mcp-docs and initialize Git
    • Create ~/playground/CodeSpace
    • Open mcp-docs Vault in Obsidian
    • Run pa init
    • Run pa index
    • Run pa mcp install
    • Restart Claude Desktop
