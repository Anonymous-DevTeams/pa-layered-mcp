    워크스페이스 구성#

    PA-Layer 시스템의 효율적인 작업을 위한 디렉토리 구조를 설명합니다.

    권장 디렉토리 구조#

    ~/playground/                    # 워크스페이스 루트
├── mcp-docs/                    # ObsidianVault (문서)
│   ├── .obsidian/               # Obsidian 설정
│   ├── ports/                   # Port 문서들
│   │   ├── auth/
│   │   │   ├── L1-AuthPort.md
│   │   │   ├── L2-LoginPort.md
│   │   │   └── L2-SignupPort.md
│   │   └── users/
│   │       └── L2-ProfilePort.md
│   ├── templates/               # 템플릿
│   ├── guides/                  # 가이드
│   └── .pa-index.yaml           # 인덱스 설정
│
├── CodeSpace/                   # 코드 프로젝트들
│   ├── my-backend/              # Kotlin/Spring 프로젝트
│   ├── my-frontend/             # React 프로젝트
│   └── my-api/                  # Go API 프로젝트
│
└── bundles/                     # 공유 Bundle 저장소

~/.pa/                           # PA-Retriever 설정
├── config.yaml                  # 메인 설정
├── default.db                   # 인덱스 데이터베이스
├── keys/                        # 서명 키
└── peers.json                   # LAN 피어 캐시

    각 디렉토리의 역할#

    1. WorkDirectory (~/playground/)#

    목적: 모든 PA-Layer 관련 작업의 루트 디렉토리

    1
2
3

    # 생성
mkdir -p ~/playground
cd ~/playground

    장점:

    • 모든 프로젝트를 한 곳에서 관리
    • Git 레포지토리별로 분리 가능
    • Claude/AI 에이전트에게 명확한 작업 범위 제공

    2. ObsidianVault (~/playground/mcp-docs/)#

    목적: PA-Layer 문서 저장소 (Port, Template, Guide 등)

    1
2
3
4
5
6
7

    # 생성
mkdir -p ~/playground/mcp-docs
cd ~/playground/mcp-docs

# Git 초기화 (권장)
git init
echo ".obsidian/workspace.json" >> .gitignore

    구조 예시:

    mcp-docs/
├── ports/                  # Port 문서
│   ├── auth/               # 도메인별 분류
│   │   ├── L1-AuthPort.md
│   │   ├── L2-LoginPort.md
│   │   └── Q-AuthQuery.md
│   └── orders/
│       ├── L1-OrderPort.md
│       └── L2-CheckoutPort.md
├── templates/              # 재사용 템플릿
│   ├── tpl-port.md
│   └── tpl-api.md
├── guides/                 # 개발 가이드
│   ├── coding-style.md
│   └── review-checklist.md
└── skills/                 # AI 스킬
    └── SKILL.md

    장점:

    • Obsidian의 강력한 마크다운 편집 기능
    • Wiki 링크로 문서 간 연결 ([[L2-LoginPort]])
    • Git으로 버전 관리 및 협업
    • 실시간 미리보기 및 그래프 뷰

    목적: 실제 코드 프로젝트 저장소

    구조 예시:

    CodeSpace/
├── my-backend/             # Kotlin/Spring
│   ├── build.gradle.kts
│   └── src/
│       └── main/kotlin/
│           └── com/example/
│               ├── auth/

│               │   └── adapter/
│               │       └── AuthAdapter.kt

├── my-frontend/            # React/TypeScript
│   ├── package.json
│   └── src/
│       ├── ports/
│       │   └── auth/
│       │       └── AuthPort.ts
│       └── ...
└── my-api/                 # Go
    ├── go.mod
    └── internal/
        └── auth/
            └── port.go

    장점:

    • PA-Retriever가 코드 시그니처 자동 인덱싱
    • Port 문서와 코드 구현체 자동 연결
    • 여러 언어/프레임워크 프로젝트 통합 관리

    4. PA 설정 디렉토리 (~/.pa/)#

    목적: PA-Retriever 설정 및 데이터 저장

    1
2
3
4
5

    # 자동 생성됨 (pa init 실행 시)
ls -la ~/.pa/

# 수동 생성 (필요시)
mkdir -p ~/.pa/keys

    파일 설명:

    ~/.pa/
├── config.yaml      # 메인 설정 파일
├── default.db       # SQLite 인덱스 데이터베이스
├── default.db-wal   # WAL 모드 파일
├── keys/            # Ed25519 서명 키 쌍
│   ├── default.pub
│   └── default.key
└── peers.json       # LAN 피어 캐시

    # ~/.pa/config.yaml
active: default

workspace:
  root: ~/playground
  obsidian_vault: ~/playground/mcp-docs
  code_space: ~/playground/CodeSpace

index:
  db_path: ~/.pa/default.db
  wal_mode: true
  exclude:
    - "*.tmp"
    - ".git"
    - "node_modules"
    - ".obsidian/workspace*.json"

types:
  preset: agile  # basic, agile, design, full

profiles:
  - name: default
    db_path: ~/.pa/default.db
    docs_path: ~/playground/mcp-docs

mcp:
  servers:
    - name: claude-desktop
      enabled: true
    - name: claude-code
      enabled: true

    워크스페이스 장점#

    1. AI 에이전트와의 협업#

    사용자: "Auth 관련 Port를 모두 검색해줘"
Claude: [pa search "domain:auth type:l2"]

사용자: "LoginPort 구현체 코드를 찾아줘"
Claude: [pa code search "LoginPort"]
    • 명확한 컨텍스트: AI가 어떤 문서/코드를 참조해야 하는지 알 수 있음
    • 토큰 효율: 필요한 문서만 선택적으로 로드
    • 일관된 구조: AI가 예측 가능한 위치에서 파일 탐색

    초기 설정 체크리스트#

    • ~/playground 디렉토리 생성
    • ~/playground/mcp-docs 생성 및 Git 초기화
    • ~/playground/CodeSpace 생성
    • Obsidian에서 mcp-docs Vault 열기
    • pa init 실행
    • pa index 실행
    • pa mcp install 실행
    • Claude Desktop 재시작
