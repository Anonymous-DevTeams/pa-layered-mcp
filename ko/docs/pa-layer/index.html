    PA-Layer#

    AI 에이전트와 함께 일하기 위한 소프트웨어 아키텍처 컨벤션입니다.

    PA-Layer란?#

    PA-Layer는 Port/Adapter(Hexagonal) 아키텍처와 CQS(Command-Query Separation) 패턴을 결합한 실용적인 아키텍처 컨벤션입니다.

    PA-Layer = Port/Adapter + CQS + AI-Friendly Convention

    왜 PA-Layer인가?#

    1. AI 에이전트 친화적#

    특징 설명
    명확한 경계 Port 단위로 책임이 분리되어 AI가 컨텍스트를 쉽게 파악
    표준화된 문서 일관된 형식으로 AI가 패턴을 학습하고 예측 가능
    선택적 로딩 필요한 문서만 로드하여 토큰 효율 극대화

    2. 토큰 효율#

    기존 RAG 방식:
┌─────────────────────────────────────────┐
│ 전체 문서 임베딩 → 유사도 검색 →        │
│ 상위 K개 청크 반환 (컨텍스트 손실 가능) │
└─────────────────────────────────────────┘

PA-Layer 방식:
┌─────────────────────────────────────────┐
│ 구조화된 검색 → 완전한 문서 반환 →      │
│ 관련 문서 체인 로드 (컨텍스트 보존)     │
└─────────────────────────────────────────┘
    항목 기존 RAG PA-Layer
    검색 방식 벡터 유사도 구조화된 쿼리
    결과 형태 문서 청크 완전한 문서
    컨텍스트 부분적 완전 보존
    의존성 추적 불가 Wiki 링크로 추적

    3. 검증된 아키텍처#

    • Hexagonal Architecture: 비즈니스 로직과 인프라 분리
    • CQS 패턴: 명령과 조회의 명확한 분리
    • 레이어 구조: L1 → LM → L2로 의존성 방향 강제

    핵심 개념#

    Port와 Adapter#

             ┌─────────────┐
         │   Client    │
         └──────┬──────┘
                │
        ┌───────▼───────┐
        │     Port      │ ← 인터페이스 정의
        │  (Interface)  │
        └───────┬───────┘
                │
        ┌───────▼───────┐
        │    Adapter    │ ← 실제 구현
        │(Implementation)│
        └───────────────┘
    • Port: 비즈니스 로직의 “무엇을” 정의
    • Adapter: 기술적 “어떻게"를 구현

    레이어 구조#

    ┌─────────────────────────────────────────────┐
│                  API Layer                   │
│        (외부 인터페이스: REST, GraphQL)       │
└─────────────────────────┬───────────────────┘
                          ▼
┌─────────────────────────────────────────────┐
│              L2 Feature Layer               │
│      (복합 기능, 여러 L1 Port 조합)          │
└─────────────────────────┬───────────────────┘
                          ▼
┌─────────────────────────────────────────────┐
│                 LM Mediator                  │
│        (L1 Port 간 조율/통합 선택적)         │
└─────────────────────────┬───────────────────┘
                          ▼
┌─────────────────────────────────────────────┐
│              L1 Domain Layer                │
│    (핵심 비즈니스 로직, 단일 도메인 책임)    │
└─────────────────────────────────────────────┘
    레이어 역할 예시
    API 외부 인터페이스 REST API, GraphQL
    L2 복합 기능 조합 LoginPort, CheckoutPort
    LM L1 간 조율 (선택적) AuthCoordinator
    L1 핵심 도메인 로직 UserPort, TokenPort

    CQS (Command-Query Separation)#

    ┌──────────────────┐     ┌──────────────────┐
│     Command      │     │      Query       │
│                  │     │                  │
│ • 상태 변경      │     │ • 읽기 전용      │
│ • 부수 효과 있음 │     │ • 부수 효과 없음 │
│ • void/Result    │     │ • 데이터 반환    │
└──────────────────┘     └──────────────────┘
