    아키텍처 개요#

    PA-Layer의 핵심 아키텍처 원칙과 규칙을 설명합니다.

    Hexagonal Architecture (Port/Adapter)#

    핵심 아이디어#

    비즈니스 로직을 기술적 세부사항으로부터 분리합니다.

                 ┌─────────────────────────────┐
             │                             │
   REST ────►│ Port                        │
             │  ↓                          │
   gRPC ────►│ Domain Logic (비즈니스 핵심) │◄──── DB
             │  ↓                          │
GraphQL ────►│ Port                        │◄──── Cache
             │                             │
             └─────────────────────────────┘
             
              좌측: Driving Adapters (입력)
              우측: Driven Adapters (출력)

    PA-Layer에서의 적용#

    우리는 Hexagonal의 완전한 구현이 아닌, 핵심 개념만 취합니다:

    원본 Hexagonal PA-Layer 적용
    Port (Interface) Port 문서 - 인터페이스 명세
    Adapter (Implementation) 코드 구현체 - Kotlin/TS/Go
    Driving/Driven 구분 입력/출력 방향 으로 단순화

    CQS (Command-Query Separation)#

    원칙#

    Command Query
    상태를 변경한다 상태를 읽는다
    부수 효과가 있다 부수 효과가 없다
    보통 반환값이 없거나 Result 데이터를 반환한다

interface UserPort {
    fun createUser(request: CreateUserRequest): User  // 생성하면서 반환

// ✅ 권장 패턴: 분리
interface UserCommandPort {
    suspend fun createUser(command: CreateUserCommand): CreateUserResult
}

interface UserQueryPort {
    suspend fun findUser(query: FindUserQuery): User?
    suspend fun listUsers(query: ListUsersQuery): List<User>
}

    레이어 규칙#

    의존성 방향#

    상위 레이어 → 하위 레이어 (단방향)

  API
   ↓
  L2
   ↓
  LM (선택)
   ↓
  L1

    규칙:

    • L1은 다른 L1에 의존할 수 없음
    • L2는 L1 또는 LM에 의존
    • LM은 L1에만 의존
    • API는 L2에 의존

    L1 (Domain Layer)#

    특징:

    • 단일 도메인 책임
    • 순수 비즈니스 로직
    • 외부 의존성 최소화

# L1-UserPort

## 책임
- 사용자 생성/수정/삭제
- 사용자 검증 로직

## 제약
- 다른 도메인 Port에 직접 의존 금지
- DB/캐시 구현체 직접 참조 금지

    L2 (Feature Layer)#

    특징:

    • 여러 L1 Port 조합
    • 사용자 기능 단위
    • 트랜잭션 경계 관리

# L2-LoginPort

## 책임
- 로그인 프로세스 전체 조율
- 인증 → 토큰 발급 → 세션 생성

## 의존성

## AI 에이전트 활용

### 문서 검색

    “auth 도메인의 L2 Port를 모두 찾아줘” → pa search “type:l2 AND domain:auth”

    “로그인과 관련된 모든 문서를 보여줘” → pa search “login”

    
### 코드 생성

    “L2-LoginPort 명세를 보고 Kotlin 구현체를 만들어줘”

    AI는:

    1. 문서 조회
    2. 의존성 확인 ([[L1-UserPort]] 등)
    3. 의존 문서도 로드
    4. 전체 컨텍스트로 코드 생성
    
### 문서 작성

    “새로운 L2-PaymentPort 문서를 기존 L2 형식으로 작성해줘”

    AI는:

    1. 기존 L2 문서 검색
    2. 패턴 학습
    3. 일관된 형식으로 새 문서 생성
