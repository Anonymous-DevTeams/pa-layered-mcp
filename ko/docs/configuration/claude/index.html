    Claude 연동#

    Claude Desktop 및 Claude Code와 PA-Retriever를 연결합니다.

    MCP (Model Context Protocol)#

    MCP는 Claude가 외부 도구와 통신하는 프로토콜입니다. PA-Retriever는 MCP 서버로 동작하여 Claude에게 문서 검색/조회 기능을 제공합니다.

# Claude Code만

# 설정 확인
pa mcp status

    연결 확인#

    1. Claude Desktop 재시작#

    설정 후 Claude Desktop을 완전히 종료하고 다시 시작합니다.

    2. MCP 도구 확인#

    Claude와의 대화에서:

    "PA-Retriever에서 사용 가능한 도구를 알려줘"

    예상 응답:

    PA-Retriever MCP 서버에서 다음 도구를 사용할 수 있습니다:

1. search_docs - 문서 검색
2. get_doc - 문서 조회
3. find_port - Port 찾기
4. get_architecture - 아키텍처 규칙
5. list_by_type - 타입별 목록
6. get_stats - 통계

    3. 검색 테스트#

    "auth 도메인의 L2 Port들을 검색해줘"

    Claude가 PA-Retriever를 호출하여 결과를 반환합니다.

    MCP 도구 설명#

    search_docs#

    문서를 검색합니다.

    사용자: "로그인 관련 문서를 찾아줘"
Claude: [search_docs query="login"]

    파라미터:

    • query: 검색어 (AND/OR/NOT 지원)
    • filters: type, domain, status, tags
    • limit: 결과 수 제한

    get_doc#

    특정 문서의 전체 내용을 조회합니다.

    사용자: "L2-LoginPort 문서 전체를 보여줘"
Claude: [get_doc id="auth.L2-LoginPort"]

    find_port#

    Port 이름으로 문서와 연결된 코드를 찾습니다.

    사용자: "LoginPort의 명세와 구현체를 찾아줘"
Claude: [find_port name="LoginPort"]

    get_architecture#

    PA-Layer 아키텍처 규칙을 조회합니다.

    사용자: "PA-Layer의 레이어 규칙을 알려줘"
Claude: [get_architecture]

    Claude와 함께 작업하기#

    문서 탐색#

    "현재 인덱싱된 문서 통계를 보여줘"
"type:l2인 문서 목록을 보여줘"
"auth 도메인의 모든 Port를 찾아줘"

    문서 작성 지원#

    "L2-PaymentPort 문서를 작성해줘. 기존 L2 문서 형식을 참고해서"

Claude는:
1. 기존 L2 문서 검색 [search_docs type:l2 limit:3]
2. 형식 분석
3. 새 문서 작성

    코드 생성#

    "L2-LoginPort 명세를 보고 Kotlin 구현체를 만들어줘"

Claude는:
1. Port 문서 조회 [get_doc id="auth.L2-LoginPort"]
2. 명세 분석
3. 코드 생성

    문제 해결#

    MCP 연결 안됨#

12

    # 1. 설정 파일 확인
cat ~/Library/Application\ Support/Claude/claude_desktop_config.json

# 2. PA 바이너리 경로 확인
which pa

# 3. MCP 서버 수동 테스트
pa serve --mcp
# Ctrl+C로 종료

# 4. 재설정
pa mcp install

    도구가 표시되지 않음#

    1. Claude Desktop 완전 종료 (Dock에서 우클릭 → 종료)
    2. 다시 시작
    3. 새 대화 시작
